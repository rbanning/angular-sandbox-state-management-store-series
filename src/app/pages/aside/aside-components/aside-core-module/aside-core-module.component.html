In all of our applications, we have a single core module (CoreModule) that is imported into the AppModule (boot module). Essentially, the core module contains all of the things that need to be available to the application when it starts. For us, these usually include the following:

Components

Any component that is part of the main application, but not referenced in any other module is declared in the core module. This usually includes header and footer components which are used in AppComponent (the component called in index.html).

Singleton Services

We put all of our singleton services into the core module. In the old days when services had to be included the "providers" section of a module, it made it easy to use the core module for declaring all of the singleton services. Now, with the "providedIn" property of @Injectable() decorator, services could be put into their own folder and don't need to be in a module. But we like the idea of keeping them in the CoreModule as both are only instantiated once.

Guards

We also put all of guards into the core module. Like services, we could move guards to their own folder and not include them in a module. But we like remain consistent and keep any "providedIn: 'root'" injectables in the CoreModule.

We use a guard to ensure that the CoreModule is only imported into the AppModule. To learn more about how we do this, check out Dan Wahlin's "Angular Jump Start" course/repo
